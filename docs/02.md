# PRD: Code Audit & Stability Fixes

## 1. Objective
Identify and repair logical errors in the "H-Tariff Tracker" application. The priority is to ensure accurate cost calculations and system stability. Existing data structures must be preserved.

## 2. Known/Suspected Issues to Investigate

### 2.1. The "First Reading" Crash
* **Context:** When the database is empty, or a user adds the very first reading.
* **Potential Bug:** The system might try to fetch `previous_reading` and fail (returning `None`), causing a crash when trying to subtract values.
* **Requirement:** Handle `DoesNotExist` or `None` gracefully. If no previous reading exists, consumption (delta) should be 0.

### 2.2. Tiered Pricing Calculation (The "Quota Split")
* **Context:** Summer consumption exceeds the yearly threshold (e.g., 2523 kWh).
* **Potential Bug:** The logic might apply the "High Price" to the *entire* month's consumption instead of just the portion that exceeded the limit.
* **Requirement:** Verify the math:
    * *Example:* Quota remaining = 50 kWh. Consumption = 150 kWh.
    * *Correct:* 50 kWh @ Low Price + 100 kWh @ High Price.
    * *Incorrect:* 150 kWh @ High Price.

### 2.3. Negative Consumption Validation
* **Context:** User accidentally types a meter value lower than the previous month.
* **Potential Bug:** The system calculates negative consumption and negative cost (paying the user).
* **Requirement:** Add validation in `forms.py` or `models.py` (clean method) to reject values lower than the previous reading.

### 2.4. Price Snapshotting Integrity
* **Context:** User changes the global price settings.
* **Potential Bug:** Old records update their calculated cost dynamically because they rely on the global settings instead of saved instance fields.
* **Requirement:** Verify that `MeterReading` saves the prices into `locked_price` fields at the moment of creation and *never* looks at the global config again for calculation.

## 3. Mandatory Test Suite (You must implement these)

Please create a `tests.py` file and implement the following test cases before fixing the code:

1.  **`test_first_reading_handling`**: Add a reading to an empty DB. Assert no crash, consumption = 0.
2.  **`test_simple_consumption`**: Add Reading A (1000) and Reading B (1100). Assert Consumption = 100.
3.  **`test_tiered_pricing_split`**:
    * Set Threshold = 2500.
    * Set Previous Cumulative = 2480.
    * Add Reading with Delta = 50 (Total = 2530).
    * Assert that 20 units are calculated at Low Price, and 30 units at High Price.
4.  **`test_price_change_isolation`**:
    * Create a reading with Price A.
    * Update Global Config to Price B.
    * Assert the reading's cost remains calculated with Price A.

## 4. Deliverables
1.  A report of found bugs.
2.  The `tests.py` file with the test cases.
3.  The corrected `models.py` and `views.py`.
