# Product Requirements Document (PRD): H-Tariff Tracker with Tiered Pricing

## 1. Project Overview
Build a Django web application to track electricity consumption for a "H-Tariff" meter. The system handles dual registers (Winter/Summer) and complex **tiered pricing for the Summer period** (quota system: usage over 2523 kWh/year is more expensive).

## 2. Core Features

### 2.1. Settings (Tiered Configuration)
* Store the following parameters (editable):
    * `winter_price`: Fixed price for heating season.
    * `summer_price_low`: Discounted price (below quota).
    * `summer_price_high`: Market price (above quota).
    * `summer_quota_threshold`: The kWh limit (e.g., 2523).
* **Quota Reset:** The system needs to know when the "Year" starts. Default to January 1st (or handle as a rolling year if simpler, but Calendar Year is preferred for MVP).

### 2.2. Meter Readings & Snapshotting
* User inputs: Date, Summer Register, Winter Register.
* **Snapshotting:** The system must save the *entire pricing structure* active at the time of reading into the database record (all 3 prices + threshold).

### 2.3. Advanced Calculation Logic (The "Split")
* **Winter Cost:** `Winter Delta * Winter Price`.
* **Summer Cost:**
    1.  Calculate `Summer Delta` (Current - Previous).
    2.  Calculate `Yearly Cumulative Summer Usage` (Sum of all summer deltas in current year *before* this reading).
    3.  **Check Quota:**
        * **Case A (Fully Below):** (Cumulative + Delta) <= Threshold $\rightarrow$ All delta * low price.
        * **Case B (Fully Above):** Cumulative >= Threshold $\rightarrow$ All delta * high price.
        * **Case C (Crossing):** Cumulative < Threshold but (Cumulative + Delta) > Threshold.
            * *Step 1:* Calculate how much fits in the remaining quota (`remaining = Threshold - Cumulative`).
            * *Step 2:* `Cost Low` = `remaining` * `low price`.
            * *Step 3:* `Cost High` = (`Delta - remaining`) * `high price`.
            * *Total Summer Cost* = `Cost Low` + `Cost High`.

## 3. Data Models (Schema Specification)

### A. `TariffConfig` (Singleton)
* `winter_price`: Decimal
* `summer_price_low`: Decimal (e.g., ~36 Ft)
* `summer_price_high`: Decimal (e.g., ~70 Ft)
* `summer_threshold`: Integer (e.g., 2523)

### B. `MeterReading`
* `reading_date`: Date
* `summer_value`: Integer (Input)
* `winter_value`: Integer (Input)
* `summer_usage`: Integer (Calculated Delta)
* `winter_usage`: Integer (Calculated Delta)
* **Snapshotted Pricing (Immutable History):**
    * `locked_winter_price`: Decimal
    * `locked_summer_price_low`: Decimal
    * `locked_summer_price_high`: Decimal
    * `locked_summer_threshold`: Integer
* `total_cost`: Decimal (Calculated using the Split Logic)
